# Evaluating OntoNG for Biomedical Ontology Generation

This repository contains the working material for a Master’s Thesis focused on evaluating ontology generation from biomedical datasets using large language models (LLMs), comparing **non-RAG** and **RAG-based** settings.

## Project goal

The goal of this project is to evaluate the ability of an LLM-based ontology generation pipeline to:

1. infer an ontological schema from biomedical tabular datasets,
2. generate an ontology in RDF/OWL/Turtle format,
3. compare the generated output with manually curated reference schemas or ontologies,
4. assess whether the use of **Retrieval-Augmented Generation (RAG)** with ontology-based context improves the results compared with a **non-RAG** setup,
5. analyze the degree of reuse or alignment with the vocabulary currently used in **BioGateway**.

## General approach

The work is structured as an experimental comparison between different conditions:

- **without RAG**
- **with RAG**, using ontology context loaded into the retrieval backend

The experiments are designed to evaluate:
- the conceptual structure inferred from each dataset,
- the classes, properties, and relations generated by the system,
- the degree of similarity or alignment with the ontologies used as reference context,
- and the structural/quality characteristics of the generated ontologies.

## Software environment

The current experimental environment is based on:

- **OntoNG** as the main ontology generation engine
- **RAGannotationAPI** as the RAG backend
- **Neo4j** for storing ontology embeddings
- **OpenAI GPT models** for schema and ontology generation
- **QASAR**, through the OntoNG plugin, for ontology quality evaluation

## Ontology context used in RAG

The RAG backend is being populated with BioGateway ontologies provided during the project. These include, among others:

- `crm`
- `crm2gene`
- `crm2phen`
- `crm2tfac`
- `gene`
- `gene2phen`
- `ortho`
- `prot`
- `prot2bp`
- `prot2cc`
- `prot2mf`
- `prot2prot`
- `reg2targ`
- `tad`
- `tfac2gene`

In addition, example files associated with some ontologies are available and may later be used for **one-shot** or **few-shot** experiments, while the main baseline experiments are intended to remain as close as possible to a **zero-shot** setting.

## Datasets explored so far

### 1. DiseaseEnhancer
A first real experiment was conducted using a 25-row sample extracted from:

- `enh2disease-1.0.2.txt`

This file comes from **DiseaseEnhancer**, a resource containing enhancer–disease associations.

### 2. TAD annotations
A second experiment was built from TAD annotation files extracted from a compressed archive. A combined CSV dataset was generated from the following files:

- `HiC_GM12878_DI_10kb.txt`
- `HiC_HMEC_DI_10kb.txt`
- `HiC_K562_DI_10kb.txt`

These files were transformed into a structured tabular dataset (`tad_3files.csv`) to be used as input for OntoNG.

## Current experimental lines

The repository is currently oriented around the following lines of work:

1. **Pilot execution of OntoNG without RAG**
   - validation of the basic pipeline on small tabular datasets

2. **Initial experiments on real biomedical data without RAG**
   - generation of ontologies from real datasets
   - inspection of the resulting diagrams and Turtle ontologies

3. **Experiments with RAG**
   - loading BioGateway ontologies into the RAG backend
   - testing whether OntoNG reuses or aligns better with expected classes and properties

4. **Ontology quality evaluation**
   - execution of the **QASAR** plugin on generated ontologies
   - inspection of ontology metrics, issues, and general quality indicators

5. **Evaluation automation**
   - ongoing work on automating the comparison between generated ontologies and the ontology vocabulary used in BioGateway

## Repository structure

A suggested repository organization is:

- `experiments/` — description of experiments and their conditions
- `data/` — transformed datasets and selected input samples
- `scripts/` — Python scripts used to preprocess data and run OntoNG
- `results/` — generated `.puml`, `.ttl`, `.json`, and evaluation outputs
- `notes/` — observations, debugging notes, and methodological decisions

## Current status

At this stage, the pipeline is already functional end-to-end for:

- ontology generation without RAG,
- ontology generation with RAG,
- and ontology quality assessment with QASAR.

The current focus is on:
- expanding the ontology context loaded into the RAG backend,
- running systematic experiments on real datasets,
- and improving the evaluation procedure in order to compare generated ontologies against the vocabulary used in BioGateway.

## Reproducibility notes

Some datasets require preprocessing before being used by OntoNG. In particular, compressed or raw text resources may need to be transformed into structured CSV files before running the generation scripts.

The exact scripts used for each experiment should be stored together with the outputs and a short description of the experimental condition.

## Important note

This repository is intended as a research and experimentation workspace for the thesis. The structure, scripts, and documented experiments may evolve as additional datasets, ontologies, and evaluation steps are incorporated.
